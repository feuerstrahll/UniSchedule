# Пояснение к проекту

## Тема

Приложение `Расписание ФА` реализовано вместо семинарских заданий по React как итоговое Flutter-приложение по теме расписания.

## Что было сделано

1. Собрано законченное Flutter-приложение для поиска расписания Финансового университета.
2. Реализован поиск групп и преподавателей через API `ruz.fa.ru`.
3. Реализован переход от результата поиска к экрану расписания.
4. Реализована загрузка расписания по выбранной группе или преподавателю.
5. Добавлен просмотр расписания на неделю.
6. Добавлена навигация по неделям и выбор даты через календарь.
7. Добавлено два представления расписания:
   - список с разделением занятий по дням;
   - табличный вид на основе локального виджета `time_scheduler_table`.
8. Добавлена обработка состояний загрузки, ошибки и пустого расписания.
9. Интерфейс оформлен в минималистичном стиле с мягкой голубой палитрой.
10. Удалены неиспользуемые учебные примеры и лишние build-артефакты.

## Соответствие Roadmap

Материалы преподавателя содержали не список отдельных экранов, а этапы, из которых собирается итоговое приложение:

- `statefull/statefull` — использовано состояние StatefulWidget для поиска, загрузки и переключения недель;
- `provider` — задача передачи выбранных данных между виджетами решена через параметры экранов и `go_router`, без глобального состояния для одного выбранного объекта;
- `news_list` и `netbloc` — реализована загрузка данных через API;
- `time_scheduler_table` — подключен локальный виджет табличного расписания;
- `flbloc` — реализованы явные состояния интерфейса: загрузка, ошибка, пустой результат и отображение данных; логика готова к дальнейшему выносу в отдельный BLoC при расширении проекта;
- `searchable_list` — реализован поиск групп и преподавателей;
- `go_router_example` — реализована маршрутизация между поиском и расписанием;
- `furshed` — доведен до релизного приложения.

## Основные файлы

- `lib/main.dart` — основной код приложения;
- `packages/time_scheduler_table` — локальный пакет табличного виджета расписания;
- `test/widget_test.dart` — smoke-тест главного экрана;
- `README.md` — инструкция по установке и запуску.

## Работа с API

Используются endpoints:

```text
GET https://ruz.fa.ru/api/search?type=group&term=...
GET https://ruz.fa.ru/api/search?type=person&term=...
GET https://ruz.fa.ru/api/schedule/group/{id}?start=yyyy.MM.dd&finish=yyyy.MM.dd
GET https://ruz.fa.ru/api/schedule/person/{id}?start=yyyy.MM.dd&finish=yyyy.MM.dd
```

Особенность API: поиск может возвращать группы или преподавателей, у которых нет занятий на выбранной неделе. Это не ошибка приложения. В таком случае показывается пустое состояние и можно выбрать другую неделю.

## Интерфейс

Главный экран:

- переключатель `Группы / Преподаватели`;
- поле поиска;
- список найденных результатов без повторов.

Экран расписания:

- информация о выбранной группе или преподавателе;
- диапазон выбранной недели;
- переключение недель;
- выбор недели через календарь;
- режимы `Список` и `Таблица`;
- группировка занятий по дням.
